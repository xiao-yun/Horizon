## [Spark 写入时捕获并发出 Parquet 页脚聚合指标](https://github.com/apache/iceberg/issues/16675) ⭐️ 7.0/10

Apache Iceberg 的 Spark 集成（issue #16675）提议新增一个可选机制：在写入 Parquet 文件时，捕获其页脚中的聚合物理统计信息（如行数、列统计等），并在提交操作时通过现有事件框架发出，而不持久化到表元数据。 该机制为 Iceberg/Spark 用户提供了对数据文件物理特征的实时可观测性，无需扫描数据或膨胀元数据，有助于性能调优、成本预估和作业监控，对大规模数据湖作业具有重要意义。 该功能是 opt-in 的，仅适用于 Parquet 格式；从页脚直接提取指标，避免全文件扫描；指标附加在提交事件上，不会污染 Iceberg 表的元数据；具体指标可能包括值计数、空值计数、行组大小等。

github · gtrettenero · Jun 3, 15:58

**背景**: Apache Iceberg 是一种面向大型分析表的开放表格式，会自动维护列级逻辑统计信息。Parquet 是列式存储格式，其文件页脚存储了压缩大小、行数等物理指标，但通常仅在读取时解压获取。Iceberg 的提交事件框架允许监听表的变更事件，本次提案借此在写入阶段即捕获物理指标并暴露给外部监控系统，无需额外查询或元数据存储。

Apache Iceberg 社区在提案 #16064 中提出一种规范级机制，为半结构化 VARIANT 列声明“虚拟字段”——系统自动存储已知路径的带类型元数据，使查询引擎能解析类型、下推谓词，并透明地将查询重定向到提取出的物理列，全程无需用户手动介入模式演进。 该提案有望大幅提升数据湖中半结构化数据的查询性能，简化频繁的模式管理操作，降低运维成本，对依赖 Iceberg 的现代数据湖架构和大数据生态具有重要价值。 提案目前处于设计阶段，尚未实现；它依赖 Iceberg v3 引入的 VARIANT 一流存储支持，且虚拟字段仅为元数据定义，不修改底层数据文件，需要各查询引擎适配相应逻辑才能生效。

github · jeffbuser · Apr 25, 03:00

**背景**: Apache Iceberg 是一种开源数据湖表格式，通过跟踪文件级元数据来管理大规模数据集，广泛应用于湖仓一体架构。VARIANT 是一种半结构化数据类型，可灵活存储 JSON 等嵌套数据，Iceberg 3.0 已提供对 VARIANT 的原生存储支持，但缺乏对其中已知字段的查询优化，该提案通过虚拟字段弥补了这一不足。
