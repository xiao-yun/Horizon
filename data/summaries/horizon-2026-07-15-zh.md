## [Bonsai 27B：首款可在手机上运行的 270 亿参数语言模型](https://prismml.com/news/bonsai-27b) ⭐️ 8.0/10

PrismML 宣布推出 Bonsai 27B，这是首个可在智能手机上运行的 270 亿参数级语言模型，采用三值量化技术，以约 4GB 大小实现接近全精度模型的性能，并以 Apache 2.0 许可证开源。 这一突破使大语言模型能脱离云端在边缘设备上运行，有望推动离线 AI 助手、隐私保护应用等场景，并可能影响苹果等公司对设备端 AI 的战略布局（据报道苹果正与 PrismML 洽谈）。 Bonsai 27B 基于 Qwen3.6-27B 进行三值量化，文件大小仅约 4GB，支持 262K 上下文长度和推测解码加速；在数学（93.40）、编程（85.96）和工具调用（74.01）等评测中保持较高性能，但视觉能力（65.19）相对较低；其存储密度是传统 IQ2_XXS 量化的两倍，为全精度（FP16）密度的近 8 倍。

hackernews · xenova · Jul 14, 17:50 · [社区讨论](https://news.ycombinator.com/item?id=48910545)

**背景**: 量化是降低神经网络数值精度的技术，例如将 32 位浮点权重转为 8 位整数，以压缩模型体积并加快推理速度，适用于手机等资源受限设备。三值量化（Ternary）更进一步，将权重限制为三个状态（如 -1、0、1），极大减小内存占用，但通常会导致性能显著下降。PrismML 采用独特的三值训练策略（知识蒸馏与量化感知训练结合），使 Bonsai 27B 在低比特下仍保有较强能力。

## [Linux 输入延迟深度评测：X11 对比 Wayland、VRR 与 DXVK 表现](https://marco-nett.de/blog/measuring-input-latency-on-linux-x11-vs-wayland-vrr-dxvk/) ⭐️ 8.0/10

一项新研究使用 500Hz 显示器系统性测量了 Linux 下不同图形栈配置的输入延迟，包括 X11、Wayland、VRR 开关以及 DXVK 转换层的影响，发现各方案存在微妙差异，例如 XWayland 比原生 Wayland 多约 3 毫秒延迟。 该分析为 Linux 桌面和游戏的输入延迟提供了难得的实测数据，有助于用户根据延迟表现选择最佳配置，并可能推动图形软件和发行版的针对性优化，对提升整个 Linux 生态的响应体验具有重要意义。 测试倚赖 500Hz 超高刷新率显示器，可能掩盖了在 60Hz 或 120Hz 等常规刷新率下出现整帧滞后的性能问题；XWayland 额外增加的 3 毫秒延迟或许正好对应一次垂直同步周期，社区建议补充低刷新率测试。作者特意设计了排除主观预判的方法，但讨论中也有观点认为延迟的‘手感’本就包含主观心理因素。

hackernews · hoechst · Jul 14, 16:36 · [社区讨论](https://news.ycombinator.com/item?id=48909424)

**背景**: X11 和 Wayland 是 Linux 上两种主要的显示服务器协议，X11 历史悠久但架构较复杂，Wayland 设计更精简、现代，理论上能实现更低延迟。VRR（可变刷新率）指显示器动态匹配 GPU 输出帧率的技术，可减少画面撕裂和卡顿。DXVK 是一个开源转换层，它能将 Direct3D 8/9/10/11 的 API 调用转译为 Vulkan，是 Proton 在 Linux 上流畅运行 Windows 游戏的关键组件。
