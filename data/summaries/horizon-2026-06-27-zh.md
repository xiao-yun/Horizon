## [Iceberg Kafka Connect 提议增加背压机制防止消息爆炸](https://github.com/apache/iceberg/issues/16389) ⭐️ 6.0/10

Apache Iceberg 的 Kafka Connect 组件提出了一项新提案（#16389），建议在 Worker 和 Coordinator 之间增加背压控制机制，当 Coordinator 过载时 Worker 能够检测其进度并自动暂停，以遏制控制主题（control topic）消息的指数级增长。 该提案直击分布式系统中常见的协调器过载问题，有助于提升 Iceberg Kafka Connect 的稳定性和可扩展性，保证数据管道在大规模负载下的顺畅运行。 提案设想 Worker 增加协调器进度检测功能，一旦发现协调器处理缓慢或超载，Worker 将主动暂停自身任务，减少对协调器的请求压力，但尚未提供具体实现细节或社区验证。

github · HenryCaiHaiying · Jun 2, 00:01

**背景**: Apache Iceberg 是一种高性能开放表格式，用于数据湖中的大型分析表管理。Kafka Connect 是 Apache Kafka 的集成框架，用于在 Kafka 与外部系统之间流式传输数据。Iceberg Kafka Connect 是连接器，将从 Kafka 主题的数据写入 Iceberg 表。在分布式模式下，Coordinator 负责分发任务，Worker 执行任务；它们通过 Kafka 控制主题进行通信，过量的控制消息可能导致协调器崩溃。
