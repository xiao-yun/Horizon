## [Nub：一款面向 Node.js 的 Bun 式全能工具包](https://github.com/nubjs/nub) ⭐️ 8.0/10

Nub 发布，利用 Node.js 的 --require 预加载钩子，以纯附加方式提供转译（基于 oxc）、模块解析及 polyfill 注入，为 Node.js 带来类似 Bun 的一体化开发体验。 Nub 让 Node.js 开发者无需切换运行时即可享受 Bun 式的便捷功能（如 TypeScript 转译、Worker polyfill），以极小开销提升了开发体验，对现有 Node.js 生态影响深远。 Nub 采用 stock node 并借助 --require 钩子加载原生 Node-API 插件（oxc 转译器），同时注册 module resolution 钩子；但社区对 ESM 支持（如顶层 await）存在疑虑，且生产环境中的性能开销和攻击面尚需评估。

hackernews · colinmcd · Jun 24, 14:14 · [社区讨论](https://news.ycombinator.com/item?id=48660267)

**背景**: Bun 是使用 Zig 语言编写的 JavaScript 运行时，基于 JavaScriptCore 引擎，内置打包、转译、任务运行和 npm 客户端，号称一体化工具。Node.js 通过 --require 和 module.registerHooks 等钩子允许在模块加载时注入自定义逻辑。Polyfill 可在低版本 Node.js 中模拟 Worker、Temporal 等较新的 Web API。

## [Apache Iceberg Kafka Connect 背压机制提案](https://github.com/apache/iceberg/issues/16389) ⭐️ 6.0/10

在 Apache Iceberg 的 Kafka Connect 中，提出一种简单的背压控制机制：让 Worker 增加对 Coordinator 进度的检测，当 Coordinator 过载时主动暂停自身，以防止控制主题消息的指数级增长。 该提案能提升 Iceberg 与 Kafka Connect 集成方案在大数据环境下的可靠性和可扩展性，避免因 Coordinator 过载导致的系统故障，尤其对高吞吐量的生产环境意义重大。 目前仅是提案阶段，具体实现细节尚未公布。核心思路是在 Worker 端引入 Coordinator 进度感知与自暂停逻辑，以轻量级方式缓解控制主题的消息积压问题。

github · HenryCaiHaiying · Jun 2, 00:01

**背景**: Apache Iceberg 是一种开源高性能表格式，用于管理分析型大数据。Kafka Connect 是 Apache Kafka 的数据流集成框架，用于在 Kafka 与外部系统间可靠地传输数据。在 Iceberg 的 Kafka Connect 中，Coordinator 通过控制主题（control topic）向 Worker 分发任务，一旦 Coordinator 过载，控制主题中的消息可能指数级增长，影响系统稳定性。
