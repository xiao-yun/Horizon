**背景**: 甲基丙烯酸甲酯（MMA）是生产有机玻璃（PMMA）的关键单体，常温下为无色液体。在不当条件下，MMA 易发生放热聚合反应，若热量不能及时散发，会引发危险的温度压力飙升。历史上已有 Garden Grove 化学泄漏等类似事故，凸显了管理此类单体的难度。

## [Apache Iceberg 社区提议为 v3 增加行级并发](https://github.com/apache/iceberg/issues/14613) ⭐️ 7.0/10

**背景**: Apache Iceberg 是一种开放式数据湖表格式。删除向量通过在文件中标记已删除行，避免了全量重写数据文件的高昂成本；行沿袭则为每行数据提供唯一标识，便于追踪变更。两者结合能实现行级并发，即并发写入时仅冲突的行会失败，而非整个文件被冲突回滚，从而提高写入效率。

## [Iceberg REST API 拟加查询参数修剪日志避免膨胀](https://github.com/apache/iceberg/issues/15947) ⭐️ 7.0/10

Apache Iceberg 社区在#15947 提案中，为 REST 目录 API 的 loadTable 端点建议增加查询参数，允许客户端请求时修剪 snapshot-log 和 metadata-log 数组，防止因无限增长导致响应体积过大和性能下降。 此优化直接解决了长期运行或高提交频率下的表加载性能痛点，能显著降低网络传输和解析开销，对维护大型 Iceberg 数据湖的用户尤为关键。 snapshot-log 记录每次快照的 ID 与时间戳，metadata-log 记录每次元数据文件变更的路径与时间戳，两个数组当前均无界增长。新增参数预计可控制返回的日志条目数量或时间窗口，具体实现尚待社区确定。

github · laserninja · Apr 12, 02:33

**背景**: Apache Iceberg 是一种开源表格式，专为大型分析表设计，支持时间旅行和快照隔离。其表元数据中的快照日志追踪所有数据变更，元数据日志记录每次元数据文件的改写历史。由于每次提交都追加记录，若不加限制，这些日志会无限膨胀，在 REST API 响应中返回完整日志将严重拖慢表加载并浪费资源。
