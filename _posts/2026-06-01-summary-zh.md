## [1-bit 量化 Bonsai 4B 扩散模型：本地设备图像生成新突破](https://prismml.com/news/bonsai-image-4b) ⭐️ 8.0/10

一种新型 1-bit 量化技术被应用于基于 Flux.2 的 4B 参数扩散模型，将内存需求大幅降至约 1GB，使得本地消费级设备能够运行高质量图像生成，且速度与画质保持竞争力。 该进展有望让普通用户摆脱对高端 GPU 或云计算订阅的依赖，在个人电脑、笔记本甚至移动端实现隐私保护的本地 AI 图像生成，推动生成式 AI 的民主化。 模型基于 Flux.2 的校正流架构，而非传统扩散模型；1-bit 量化后推理速度略慢于原始小型 Flux.2 模型，但存储占用剧减，可在仅 1GB 显存的设备上运行。

## [Linux rseq 实现高效无锁每 CPU 并发](https://justine.lol/rseq/) ⭐️ 8.0/10

近日一篇技术文章详细探索了 Linux 内核的可重启序列 (rseq) 功能。该文章通过具体的 x86 汇编示例，演示了如何利用 rseq 实现高效的无锁每 CPU 并发编程，全程无需使用原子操作或互斥锁。 这一实践为高性能计算场景提供了全新的并发控制思路。它能够帮助开发者规避传统锁机制和原子操作的性能瓶颈，尤其适用于对延迟敏感的服务器端系统和底层库。 rseq 的核心机制是用户空间向内核注册一段通常不超过 10 条指令的临界区，若执行期间发生中断或线程迁移，内核会强制将指令指针重置到序列开头从而重试。此外，rseq 还通过共享内存让用户程序快速获取当前 CPU 编号等信息，避免了系统调用开销。

hackernews · grappler · May 31, 14:38 · [社区讨论](https://news.ycombinator.com/item?id=48346019)

**背景**: 可重启序列 (rseq) 是 Linux 4.18 内核引入的用户态同步原语，由 Paul Turner 最初提出并经 Mathieu Desnoyers 推动合入。它允许线程安全地操作每 CPU 数据而无需持有锁或使用代价高昂的原子指令，其原理类似乐观的“尝试-失败-重试”模式，但重试由内核透明完成。这项技术已被 glibc 和多个高性能库采用，用于优化 per-CPU 计数器和环形缓冲区等常见结构。

## [Apache Hudi 提出支持分区软删除功能](https://github.com/apache/hudi/issues/18774) ⭐️ 6.0/10

在 issue #18774 中，Apache Hudi 社区提出了一项功能提议，计划为分区管理增加软删除支持，允许用户在分区被标记删除后、永久清理前临时恢复数据。 该功能为数据湖的分区操作提供了安全缓冲，降低了误删带来的数据丢失风险，有助于增强数据管理的容错能力和用户信心。 软删除期间，分区的文件和索引元数据暂时保留，待清理服务在实际删除时彻底移除。目前该提议处于早期讨论阶段，尚无具体实现细节或社区反馈。

github · kbuci · May 18, 22:43

**背景**: Apache Hudi 是一个开源数据湖平台，提供 ACID 事务和高效的 upsert/delete 等数据库能力。其现有 delete_partition API 会直接替换分区文件并由清理服务永久删除，缺乏恢复机制。该提案引入软删除，在数据彻底清除前增设可恢复的缓冲期。
