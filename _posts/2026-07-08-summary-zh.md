## [腾讯开源 295B 混合专家模型 Hy3，21B 活跃参数，Apache 2.0 许可](https://simonwillison.net/2026/Jul/6/hy3/#atom-everything) ⭐️ 8.0/10

腾讯发布了 Hy3 模型，这是一个拥有 295B 总参数的混合专家（MoE）语言模型，每个 token 激活 21B 参数，性能超过同规模模型，并可与 2-5 倍参数量的旗舰开源模型竞争，采用 Apache 2.0 许可证开源。 该模型以 Apache 2.0 许可免费提供，并在 OpenRouter 上免费使用，大幅降低了高性能大模型的获取门槛，有望加速各类产品和生产力任务中的 AI 应用落地。 模型额外包含 3.8B 的 MTP 层参数，全尺寸版本 598GB，FP8 量化版本 300GB，支持 256K 上下文长度，截至 7 月 21 日可在 OpenRouter 免费调用。

rss · Simon Willison · Jul 6, 23:57

**背景**: 混合专家（MoE）是一种将 AI 模型划分为多个“专家”子模型的架构，每个专家专注于处理部分输入数据，每次推理仅激活部分专家，从而在控制计算成本的同时大幅扩展总参数规模。活跃参数指模型处理每个 token 时实际参与计算的参数量，而总参数包括所有专家的参数总和。这种设计使得模型能更高效地利用计算资源，以较小的推理成本达到接近更大规模密集模型的性能。

## [Apache Iceberg 提议增加 Flink 水印与计算列元数据支持](https://github.com/apache/iceberg/issues/16756) ⭐️ 8.0/10

Apache Iceberg 社区收到了一项提案（#16756），建议在 Iceberg 表元数据中增加对 Flink 水印与计算列的支持，以便在流式 SQL 引擎中持久化这些定义。 该改进将使 Iceberg 完整保留流式表的关键元数据，避免在跨系统或重复查询时丢失水印和计算列信息，从而提升 Iceberg 与 Flink 等流处理引擎的集成深度。 提案指出，当前 Iceberg 目录通常只保存表结构，不存储 Flink DDL 中的 WATERMARK 和计算列表达式；实现后，这些元数据将随表持久化，方便流式查询直接利用。

github · SteveStevenpoor · Jun 12, 03:53

**背景**: Apache Iceberg 是一种开放表格式，用于在数据湖上管理大规模结构化数据。水印是流处理中根据事件时间控制状态保持时长的机制，用于处理迟到数据；计算列则是从其他列派生而来的虚拟列。Flink 等流式 SQL 引擎依赖这些元数据进行窗口聚合、去重等操作。
