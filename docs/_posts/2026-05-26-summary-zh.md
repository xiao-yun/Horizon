## [DynIP：原生支持 RFC 2136、IPv6 与 DNSSEC 的动态 DNS 服务](https://dynip.dev/) ⭐️ 7.0/10

网络工程师 Daniel 推出了 DynIP 动态 DNS 服务，原生支持 RFC 2136/TSIG 更新协议、端到端 IPv6 和 DNSSEC，无需自定义客户端即可兼容 FortiGate、MikroTik 等设备。 这填补了动态 DNS 市场对现代网络协议（如 IPv6 和 DNSSEC）支持的缺口，通过标准 RFC 2136 协议为企业和家庭用户提供了更安全、更兼容的自有设备接入方案。 该服务以 RFC 2136/TSIG 更新为主要路径，同时提供 HTTP API 作为后备；已实测兼容 FortiGate genericDDNS 和 MikroTik 的/tool dns-update，并可与 Kubernetes 的 external-dns 集成。

hackernews · dynip · May 26, 07:35 · [社区讨论](https://news.ycombinator.com/item?id=48276363)

**背景**: 动态 DNS（DDNS）用于将域名自动指向 IP 地址可能经常变化的设备，传统 DDNS 服务常依赖私有 HTTP 协议且对 IPv6 和 DNSSEC 支持不足。RFC 2136 是 IETF 制定的 DNS 动态更新标准，支持 TSIG 签名认证；DNSSEC 则通过数字签名确保 DNS 应答的完整性和真实性。现代网络设备和企业环境日益需要对这些标准的原生支持。

## [Garden Grove 化学品储罐事故背后的化学原理](https://www.science.org/content/blog-post/methyl-methacrylate-tank) ⭐️ 6.0/10

一位化学博主针对近期 Garden Grove 发生的甲基丙烯酸甲酯（MMA）储罐火灾事件，详细解析了该化学品的性质与危险性。 该事件凸显了化工品储运安全的重要性，尤其是 MMA 在高温下可能发生聚合放热、导致沸液蒸汽爆炸（BLEVE）等极端风险，对工业安全规程和应急响应具有警示意义。 事故中储罐出现裂缝，意外释放了压力，避免了热失控和 BLEVE；MMA 本身易燃、具刺激性，且聚合反应放热猛烈，其急性口服毒性 LD50 为 7-10 g/kg（大鼠）。

hackernews · nooks · May 26, 19:25 · [社区讨论](https://news.ycombinator.com/item?id=48284712)

**背景**: 甲基丙烯酸甲酯（MMA）是生产有机玻璃（PMMA）等聚合物的关键单体，常温下为无色易燃液体，易挥发，具有刺激性气味。在高温或储存不当条件下，MMA 可发生聚合反应并大量放热，导致储罐压力急剧升高，引发火灾甚至爆炸。这种失控反应与 2011 年日本地震后化工厂事故类似，凸显了被动安全系统的重要性。
