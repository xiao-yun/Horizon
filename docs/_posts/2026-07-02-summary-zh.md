## [FFmpeg 9.1 发布全新 AAC 编码器，音质大幅提升](https://hydrogenaudio.org/index.php/topic,129691.0.html) ⭐️ 8.0/10

FFmpeg 9.1 版本内置了全新设计的 AAC 编码器，一举解决了过去编码器音质低劣的问题，并对 48kHz 采样率音频进行了特别优化。 AAC 编码是广泛使用的音频压缩标准，FFmpeg 作为多媒体处理核心工具，此次改进将惠及大量依赖其进行音频转码和视频制作的用户，显著提升其输出质量。 新编码器主要针对 48kHz 音频优化，其他采样率虽支持但质量非最优；同时通过规避解码器中立体声 PNS（知觉噪声整形）的 bug 来确保兼容性。

hackernews · ledoge · Jul 1, 14:10 · [社区讨论](https://news.ycombinator.com/item?id=48747116)

**背景**: FFmpeg 是一款开源跨平台的多媒体框架，常用于音视频编解码和转码。AAC（高级音频编码）是一种有损压缩音频格式，被广泛用于流媒体和存储。此前，FFmpeg 内置的 AAC 编码器因音质不佳（常出现高频嘶嘶声伪影）而备受诟病，用户往往需要额外安装 Apple Core Audio 等外部编码器以获得可接受的效果。

**社区讨论**: 社区普遍欢迎这一改进，许多人提到旧版编码器的高频嘶嘶声问题，期待新版本实际表现。也有讨论指出 Opus 在同等码率下音质更优，以及编码器调优依赖开发者主观听感的有趣现象。对于 48kHz 优化存在不同看法，有人认为这已是标准，有人质疑其普遍性。

## [Ciechanowski 发布内燃机交互式深度解析文章](https://ciechanow.ski/internal-combustion-engine/) ⭐️ 8.0/10

Ciechanowski 于 2021 年发布了一篇高度互动的技术文章，深入解析内燃机的机械结构与工作原理，以其清晰的视觉呈现和教育价值获得广泛赞誉。 该文章通过交互式动画使复杂的机械工程知识易于理解，不仅对汽车爱好者有教育意义，也引发了关于发动机设计演进与控制技术发展的深入讨论。 文章详细展示了曲轴、活塞、气门机构等核心部件的运动，并探讨了从化油器到电喷、从推杆 V8 到底置凸轮轴等设计演变，以及流体动力润滑等关键技术细节。

hackernews · StefanBatory · Jul 1, 13:04 · [社区讨论](https://news.ycombinator.com/item?id=48746076)

**背景**: 内燃机是一种将燃料化学能转化为机械能的装置，广泛应用于汽车、船舶等领域。其基本工作循环包括进气、压缩、做功和排气，关键部件有气缸、活塞、曲轴等。传统发动机依赖机械式化油器混合燃油与空气，现代则普遍采用电子燃油喷射系统，通过精确控制空燃比提升效率并减少排放。文章通过交互方式直观呈现了这些机理。

**社区讨论**: 社区评论普遍赞赏文章的清晰度和深度，有人怀念推杆 V8 发动机的简洁优雅，认为现代顶置凸轮轴等技术虽提升性能但缺乏美感；也有人强调控制系统的进步，如电喷替代化油器极大降低了污染。此外，流体动力润滑的讨论也引人深思。

ClickHouse 26.6 版本正式发布，带来了假设跳过索引、支持级联刷新的可刷新物化视图，以及实验性的持续查询功能。 这些新特性提升了数据库的查询优化和实时数据处理能力，使开发者和数据分析师能更灵活地测试索引效果、简化数据管道维护，并支持持续运行的分析查询。 假设索引是会话范围的虚拟跳过索引，无需实际构建即可通过 EXPLAIN WHATIF 评估查询性能；可刷新物化视图新增 CASCADE 选项，实现依赖视图的自动级联刷新；持续查询作为实验特性，允许 SQL 语句持续运行并实时处理输入数据。

## [Apache Iceberg v4 规范拟支持 varchar 和 char 类型](https://github.com/apache/iceberg/pull/16829) ⭐️ 6.0/10

PR #16829 提议在 Apache Iceberg v4 规范中新增 varchar(N) 和 char(N) 原始类型，以改善与传统 SQL 引擎的兼容性，Spark 及 Trino 等引擎已支持这些类型。 此举大幅提升了 Iceberg 与传统数据库的兼容性，便于数据迁移和跨系统查询，使习惯使用固定长度或可变长度字符类型的用户能无缝接入数据湖。 该提案属于 Iceberg v4 规范更新，Spark（3.1.0 起）提供 VarcharType 和 CharType，Trino 原生支持；目前仅为规范层定义，实现需各引擎适配。

github · ebyhr · Jun 17, 13:55

**背景**: Apache Iceberg 是一种开源高性能表格式，用于管理数据湖中的大型分析表，支持多引擎并发读写和 ACID 事务。其规范目前版本为 v3，v4 正在开发中。传统 SQL 数据库常用 char(n)（固定长度）和 varchar(n)（可变长度）字符串，而 Iceberg 此前主要使用 string 类型，缺乏长度约束。
