## [AWS 计费单位混淆致账单飙升至 17 亿美元](https://news.ycombinator.com/item?id=48945241) ⭐️ 8.0/10

近期，因 AWS 计费系统中的单位混淆（将字节误当作 GB），部分用户（包括一名正常月费低于 5 美元的用户）的估算账单一度飙升至 17 亿美元。 该事件暴露了云服务计费系统的脆弱性，提醒企业检查自动计费的准确性；对依赖 AWS 的开发者而言，这种错误可能导致恐慌和业务中断。 根据社区前 AWS 员工的评论，错误根源是服务发出计量值时单位未明确指定，计费系统默认以字节为单位，导致实际计费错误按“5 美分/字节”计算。

hackernews · nprateem · Jul 17, 09:42

**背景**: AWS 的计费系统由计量数据聚合产生，每个服务项目（SKU）在定价计划中定义。通常计量指标有明确单位（如 GB 或请求次数），但若定义缺失或不匹配，系统可能默认使用字节等基础单位，从而引发计算偏差。类似问题过去也曾出现过，比如早期 EC2 预留实例的节约计算错误。

**社区讨论**: 社区反应强烈：有用户调侃称收到 7800 万美元账单时受到“情感伤害”；前 AWS 员工证实了此类单位错误的内部处理流程，通常几小时内能修正；也有人提及十年前曾发现真实的计费漏洞，难以让 AWS 承认错误。总体情绪以惊讶和无奈为主，同时赞赏 AWS 能快速修复。

天文学家利用詹姆斯·韦伯空间望远镜（JWST）的发射光谱技术，在距离地球 48 光年的红矮星 LHS 1140 的宜居带内，于岩石质系外行星 LHS 1140b 上探测到大气特征，排除了其属于迷你海王星的设想。 这是人类首次在宜居带内的岩石行星上确认大气存在，标志着系外行星宜居性研究迈出关键一步，为未来寻找潜在生物标志物和评估行星可居住性提供了重要目标。 LHS 1140b 半径约为地球的 1.7 倍，通过观测其运行到恒星背后时的二次蚀发射光谱，研究人员区分了厚实的氢氦大气和较薄的大气层，确认该行星并非迷你海王星。该星围绕一颗相对不活跃的红矮星公转，轨道周期约 25 天，较强的引力可能有助于束缚住大气。

## [Firefox 被编译为 WebAssembly，可在浏览器中完整运行](https://simonwillison.net/2026/Jul/16/firefox-in-webassembly/#atom-everything) ⭐️ 8.0/10

Puter 团队将 Firefox 浏览器引擎（Gecko）编译为 WebAssembly，使其能够通过 WebSocket 协议完整运行在另一个浏览器标签页中。 这展示了将完整的桌面级应用迁移到 Web 平台的技术可行性，为跨平台兼容、安全沙箱和云端浏览器服务开辟了新可能，尽管目前尚不适合日常使用。 项目利用了 Firefox 较强的单进程支持，并通过 Wisp 协议代理所有网络请求以绕过浏览器沙箱限制；开发过程使用了价值约 25,000 美元的 AI 编程辅助（Claude Opus 和 Fable 令牌），但通过订阅计划实际花费较低。

rss · Simon Willison · Jul 16, 23:34

**背景**: WebAssembly（Wasm）是一种低级的二进制指令格式，旨在为编程语言提供可移植的编译目标，使 C/C++ 等语言编写的高性能应用程序能够在浏览器中运行。浏览器运行时沙箱通常禁止直接打开网络连接，因此需要通过 WebSocket 将流量代理至外部服务器。将完整的浏览器引擎编译为 Wasm 是一项巨大的工程挑战，因其代码量庞大且高度依赖操作系统服务。

## [应对问题的三种非解决反应：保留问题等](https://improvesomething.today/responses-to-problems/) ⭐️ 7.0/10

这篇文章指出了人们在面对问题时除了直接解决之外的三种常见反应，例如“保留问题”，并揭示了背后的系统和个人动机。 它解释了为何许多社会和组织问题长期得不到解决，因为负责解决问题的人往往缺乏真正的解决动力，对理解官僚主义和组织低效具有重要意义。 文章详细剖析了部分非解决性反应，如保留问题以维持预算或职位，并讨论了这些行为背后的心理和组织激励。

## [Apache Iceberg REST Catalog 提议添加表级标签元数据](https://github.com/apache/iceberg/issues/15521) ⭐️ 6.0/10

Apache Iceberg 社区提出在 REST Catalog 的 LoadTableResponse 中增加一个可选的 'labels' 字段，用于标准化传递表的所有权、分类等上下文元数据。 该提案填补了 REST Catalog 在多引擎间传递非技术元数据的标准缺失，有助于消除厂商锁定，让开源引擎也能消费目录中维护的标签信息，提升数据治理互操作性。 新增的 'labels' 字段为可选，允许目录服务通过标准响应体返回表级别的标签键值对，避免依赖厂商私有扩展；目前讨论限于表级，列级标签可能后续扩展。

github · laskoviymishka · May 12, 08:00

**背景**: Apache Iceberg 是一种开放表格式，其 REST Catalog 规范定义了通过 REST API 管理表元数据的标准接口。LoadTableResponse 是加载表时服务端返回的元数据载荷，目前包含 schema、快照等信息，但缺少所有权、分类等业务元数据的标准传递方式，导致不同目录实现间不兼容。
